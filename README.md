# 로또

## 진행 방법

- 로또 요구사항을 파악한다.
- 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 코드 리뷰 요청을 한다.
- 코드 리뷰 피드백에 대한 개선 작업을 하고 다시 PUSH한다.
- 모든 피드백을 완료하면 다음 단계를 도전하고 앞의 과정을 반복한다.

## 그라운드 룰

- 서로의 의견을 존중한다.
- 페어 프로그래밍 롤은 하나의 기능을 구현할 때마다 스위칭한다.
- 객체지향적인 설계를 우선시한다.

## 요구사항 정리

- [ ] 로또 한장의 가격은 1000원이다.
- [ ] 게임이 시작되면 로또 구매 가격을 입력한다.
  - [ ] 구매 가격 입력이 숫자가 아닌 경우 에러 출력을 발생시킨다.
  - [ ] 구매 최대 가격은 최소 0원, 최대 100,000,000원이다. (임의 지정)
    - [ ] 구매 가격이 음수인 경우 에러 출력을 발생시킨다.
    - [ ] 입력이 구매 최대 가격을 넘는 경우 에러 출력 발생시킨다.
  - [ ] 입력이 입력 조건에 맞지 않는 경우 에러 출력을 발생시키고 재입력을 받는다.
- [ ] 구매 가격을 입력 받고 구매한 로또의 갯수를 계산한다.
- [ ] 로또의 갯수 만큼 임의의 로또 번호를 출력한다.
  - [ ] 로또 숫자의 범위는 1 ~ 45 사이이다.
  - [ ] 한 게임당 로또 숫자의 갯수는 6개이다.
  - [ ] 각 숫자는 중복될 수 없다.
  - [ ] 각 로또 번호는 오름차순으로 출력한다.
  - [ ] 한 줄의 한 게임의 로또 번호를 출력한다.
- [ ] 사용자에게 당첨 번호를 입력받는다.
  - [ ] 당첨 번호 입력은 숫자 6개이다.
  - [ ] 각 숫자는 `, `로 구분된다.
  - [ ] 각 숫자는 1 ~ 45까지의 숫자이다.
  - [ ] 각 숫자는 중복될 수 없다.
  - [ ] 입력이 입력 조건에 맞지 않는 경우 에러 출력을 발생시키고 재입력을 받는다.
- [ ] 사용자에게 보너스 번호를 입력받는다.
  - [ ] 보너스 번호는 1개만 입력 받아야 한다.
  - [ ] 보너스 번호는 1 ~ 45 범위에 있어야 한다.
  - [ ] 입력이 입력 조건에 맞지 않는 경우 에러 출력을 발생시키고 재입력을 받는다.
  - [ ] 보너스 번호는 당첨 번호와 중복될 수 없다.
- [ ] 구매한 로또에 대한 당첨 통계를 출력한다.
  - [ ] 3개 ~ 6개 일치 갯수를 각 줄에 출력한다.
  - [ ] `5개 + 보너스번호` 일치 갯수를 출력한다.
  - [ ] 구매금액에 대한 수익률을 출력한다.
    - [ ] 소숫점 세번째 자리에서 반올림하여 수익률을 출력한다.
